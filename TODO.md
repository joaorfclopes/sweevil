## Migrate from Heroku to Vercel
Backend is Express + Mongoose running as a persistent server process — incompatible with Vercel's serverless model out of the box. Migration steps: (1) wrap `server.js` as a Vercel serverless function in `api/index.js`; (2) add `vercel.json` with rewrite rules routing `/api/*` to the function and `/*` to the built frontend; (3) handle Mongoose cold-start reconnection (can't rely on a persistent connection); (4) remove the Heroku-specific `app.set('trust proxy', 1)` or replace with Vercel's equivalent; (5) remove `Procfile` and `app.json`. Frontend static files go to Vercel CDN automatically via the build output. Stripe webhooks raw body handling is already in place. Do the async image processing Lambda (see below) before this, since sharp processing will hit Vercel's 10s serverless timeout otherwise.

## Async image processing via S3 Lambda
Currently `uploadRoute.js` POST `/s3` runs sharp synchronously inside the request (resize + AVIF encode) before returning the S3 URL. With `effort: 2` this stays under Heroku's 30s timeout, but quality is capped. Migration: (1) receive file, upload raw buffer to S3 as `pending/{folder}/{uuid}.orig` with folder encoded in object metadata, return a `{ jobId }` immediately; (2) S3 event notification triggers a Lambda on the `pending/` prefix — Lambda runs sharp at `effort: 6, quality: 65`, writes final AVIF to `{folder}/{uuid}.avif`, deletes the pending file; (3) add `GET /api/uploads/status/:jobId` backend endpoint that checks whether `{folder}/{uuid}.avif` exists in S3; (4) frontend polls that endpoint until URL is ready, then proceeds to save the gallery/product record. Affects `GalleryAdminTab.jsx` upload flow and `ProductEditScreen.jsx`. Required before Vercel migration — Vercel's 10s function timeout makes synchronous sharp encoding impossible for large images.
